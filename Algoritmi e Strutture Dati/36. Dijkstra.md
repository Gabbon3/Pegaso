# 36. Dijkstra

| Versione (Nome) | Struttura Dati usata | Complessità (Velocità) | Concetto chiave |
| --------------- | -------------------- | ---------------------- | --------------- |
| Base | Vettore (Array) | $O(n^2)$ | È la più lenta. Scansiona tutto il vettore ogni volta per trovare il minimo. |
| Johnson | Heap Binario | $O(m  log n)$ | È la via di mezzo. L'Heap Binario è un albero ordinato |  molto più veloce del vettore. |
| Fibonacci | Heap di Fibonacci | $O(m + n log n)$ | È la più veloce (la Ferrari). L'Heap di Fibonacci è una struttura complessa ottimizzata per queste operazioni. |
