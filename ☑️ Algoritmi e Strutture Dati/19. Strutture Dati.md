# 19. Strutture Dati

- insieme -> raccolta di elementi senza organizzazione
- struttura dati -> raccolta di elementi organizzata e logica

## 1. Introduzione alle strutture dati

### Strutture dati comuni

- Array: Una fila ordinata di elementi dello stesso tipo, accessibili tramite un numero (indice).
- Liste: Elementi collegati tra loro (come una catena).
- Stack (Pila): Come una pila di piatti: aggiungi e togli solo da sopra (LIFO - Last In First Out).
- Coda: Come una fila alla cassa: si entra dal fondo e si esce dalla testa (FIFO - First In First Out).
- Alberi: Organizzazione gerarchica (come l'albero genealogico), con padri e figli.
- Grafi: Nodi collegati da archi (come una mappa stradale tra città).
- Hash Table: Usa una formula matematica per trovare subito dove mettere o cercare un dato

### Tassonomia sulle strutture dati

Tassonomia classica ~ come si classificano le strutture dati?
- __statica vs dinamica__:
    - statica -> gli array, la dimensione non cambia rimane fissa
    - dinamica -> puoi aggiungere o togliere elementi (Liste)
- __compatta vs sparsa__ - come i dati sono disposti sulla memoria fisica: 
    - compatta -> gli elementi sono fisicamente vicini sulla RAM (Array classici)
    - sparsa -> gli elementi sono sparsi in giro e connessi tramite puntatori (Liste e Alberi)
- __ordinata vs non ordinata__ -> si basa sul valore delle chiavi (albero è ordinato tramite le chiavi, la hash table no)

## 2. Insieme Dinamico

1. Dizionario -> insieme dinamico che supporta operazioni di inserimento, cancellazione e verifica di appertenenza
2. Operazioni (Query Modifica) -> le operazioni si dividono in:
    - Query (Lettura):
        - `SEARCH(S, k)`: trova l'elemento con chiave k
        - `MINIMUM(S)` / `MAXIMUM(S)`: trova piccolo e grande
        - `SUCCESSOR(S, x)` / `PREDECESSOR(S, x)`: trova l'elemento immediatamente successivo o precedente.
    - Modifica (Scrittura):
        - `INSERT(S, x)`: Aggiungi elemento
        - `DELETE(S, x)`: Rimuove un elemento
3. Astratto VS Concreto:
    - __Struttura dati astratta ADT__ -> rappresenta il _cosa fa_. Un modello generico che descrive le operazioni senza dire come sono fatte.
    - __Struttura dati concreta__ -> _come lo fa_. L'implementazione specifica del codice

## 3. Il Puntatore

1. A cosa serve? I puntatori sono _fondamentali per le strutture dinamiche_
2. Sintassi C++:
    - `*` -> dichiara un puntatore o accede al valore puntato
    - `&` -> ottiene l indirizzo di memoria di una variabile
    - `new` -> alloca nell _heap_
    - `delete` -> dealloca nell _heap_
3. Python vs C++ -> in python i puntatori sono impliciti e sono gestici dal GC garbage collector. non serve fare new e delete
4. Applicazione visiva:
    - Liste -> ogni nodo ha un dato e un puntatore al prossimo nodo
    - Alberi -> ogni nodo ha un dato e puntatori ai figli (sinistro e destro)
