## Criteri di copertura

Come misuri se hai testato abbastanza?

1. Statement Coverage - minimo sindacale
    - devi eseguire ogni riga di codice almeno 1 volta
    - problema: è un modello molto debole
2. Branch Coverage - standard
    - in ogni if testi sia condizione vera che falsa
    - considerato miglior compromesso
3. Condition Coverage - copertura condizioni
    - va piu a fondo di branch coverage, ad esempio se una if è articolata: if (A > 0 AND B < 100) controlla A e B
4. Multiple Condition Coverage - Incubo
    - testi tutte le combinazioni possibili
    - precisissimo ma si torna nel problema dei grandi numeri, diventa impraticabile testare tutto.
    - usato solo per sistemi davvero critici come aerei o medicina
5. Path Coverage - La teoria
    - TEsto ogni possibile strada da inizio a fine programma
    - è impossibile se ci sono loop perche le combinazioni sono infinite

## Tecniche Avanzate

come capire se i test sono buoni:

- __Mutante__ -> modifica voluta errata sul codice
    - se i test con il mutante passano tutti allora errore

questo di chiama __Mutation Testing__
