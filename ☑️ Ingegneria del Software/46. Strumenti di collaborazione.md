## Github Git

- Pull request -> spazio di confronto tecnico, Code Review
- Code Review -> serve a condividere la conoscenza e a trovare possibili bug

## Continous Integration CI

- definizione -> pratica di integrazione frequente e incrementale
- contrasto con passato -> approccio a blocchi generava merge hell, il ci invece permette integrazioni frequenti, merge contenuti e semplici.
- __Pipeline__ standard:
    1. Build (compilazione)
    2. Test Automatici (unit test, integration test)
    3. Analisi Statica (stile, bug)
