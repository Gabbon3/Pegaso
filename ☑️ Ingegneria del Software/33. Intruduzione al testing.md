# 33. Introduzione al Testing

- __Obiettivo__: dimostrare che il software non funziona
- __Mentalita__: il testing è distruttivo, devi cercare di rompere il software
- __Paradosso__: un test di successo è un test che fallisce perche ha trovato un bug o un comportamento inatteso.

## 4 livelli di testing

- __Unit testing__: test su singoli pezzi, isolato dal resto
- __Integration testing__: test su piu pezzi insieme che collaborano
- __System testing__: testi l intero sistema, simulando utilizzo reale
- __Acceptance testing__: lo fa il cliente. serve confermare "si è quello che volevo".

## Non solo funzionalità

bisogna testare come il software fa:
- __Performance Testing__: il sito regge migliaia di richieste al secondo?
- __Usability Testing__: test sulla UX
- __Security Testing__: è sicuro?

## Panoramica

### Ciclo di vita del test

5 fasi:
1. __Pianificazione__: decidi cosa testare e chi lo fa, definisci le priorità
2. __Progettazione__: scrivi i test case, definisci input e risultati attesi
3. __Esecuzione__: lanci i test e raccogli risultati
4. __Analisi__: se ce un errore capisci se è colpa del codice o del test malscritto
5. __Manutenzione__: aggiorni i test quando il software cambia

## Retesting VS Regression

il _Retesting_ (riesecuzione) hai trovato il bug sul flusso x e ritesti il flusso x.
il _Regression_ (regressione) è quando testi anche vecchi flussi per vedere se il nuovo codice non abbia effetti collaterali

## Ruoli

- __Sviluppatore__: fa test unitari Unit Test
- __Tester__: progetta i test di sistemi Integration Test
- ___Analista__: verifica che i requisiti siano testabili System test
- __Cliente__: fa i test di accettazione Acceptance test

## Testing nel ciclo di sviluppo

menzione a varie tecniche come waterfall, agile...

## Concetti chiave

### Triade del Male

- Errore -> umano, lo sbaglio logico nel programmatore
- Difetto -> codice, bug scritto nel file
- Fallimento -> realta, crash o calcolo errato che vede l utente

### Oracolo

Per sapere se un test è passato o fallito serve un oracolo.

Puo essere un documento, un calcolo matematico o un altro software che dice semplicemente "il risultato doveva essere x"

### Testing vs Debugging

- testing: ti dice che ce un problema
- debugging: ti dice perche e lo risolve
