## 3 Leggi di Uncle Bob

Robert C. Martin ~ Uncle Bob:
1. non scrivere codice di produzione se non hai un test fallito
2. non scrivere piu test di quanto serve per fallire
3. non scrivere piu codice di quanto serve per passare il test

## Come scrivere buoni test (FIRST)

per evitare di scrivere test inutili si consiglia di seguire questo flusso:

- __F__ ast -> devono girare in ms
- __I__ ndipendent -> il test A non deve dipendere dal risultato di B
- __R__ epeatable -> deve dare lo stesso risultato su qualsiasi ambiente viene eseguito
- __S__ elf-validating -> deve saper dire "Passato" "Fallito" fine.
- __T__ horough -> deve coprire anche i casi limite (errori, null...)
