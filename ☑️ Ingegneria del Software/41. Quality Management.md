# 41 Quality Management

## Intruduzione e definizioni Chiave

1. Definizione moderna di qualita -> cose che rendono conforme il software a:
    - Requisiti formali: quelli indicati nel contratto
    - Aspettative degli utenti: quello che effettivamente l utente si aspetta

### 3 livelli operativi

1. organizzativo -> azienda definisce standard per tutti i progetti
2. livello di progetto -> il team adatta gli standard aziendali al progetto
3. pianificazione qualita -> si definiscono obiettivi precisi e come verificarli.

### Qualita del processo / Qualita del prodotto

- Avere un buon processo aiuta ma non garantisce qualita assoluta
- il software è un artefatto cognitivo -> l eccesso di burocrazia puo uccidere la qualita

### Soggettività

Molti attributi come leggibilita e manutenibilita non sono misurabili a righello. Serve un __giudizio esperto__

## Qualita del Software e Standard

### 2 tipi di standard

- Standard di Prodotto -> definiscono come deve essere fatto l output
- Standard di Processo -> definiscono come si deve lavorare

### ISO 9001

la iso non è specifica per il software, è piu generica.

__non garantisce__: che il software sia bello o senza bug
__garantisce__: che l azienda ha un metodo documentato e tracciabile

_in sintesi_: anche se l azienda è certificata non garantisce che scriva codice buono.

### Il Rischio Burocrazia

troppi standard uccidono l innovazione:
- se imponi regole rigide senza motivo il team si blocca -> _Burocrazia Sterile_
- Soluzione -> usare strumenti automatici come Linter, Validatori, Template per rispettare gli standard senza pesare troppo

## Revisioni e Ispezione del Codice

### Statica VS Dinamica

- Testing -> dinamica: esegui il codice e vedi se funziona
- Revisione/Ispezione -> statica: NON esegui il codice. Lo leggi e analizzi a mente fredda, senza debug o altro, guardi la doc.

### Perche farla?

Le ispezioni trovano piu spesso difetti che nel testing:
- è una tecnica __preventiva__ trovi l errore mentre stai scrivendo o subito dopo
- Regola _Empirica_: correggere un bug subito costa 10€, correggerlo in prod costa 1000€ o peggio  (cliente perso)

### I ruoli dell ispezione formale

l ispezione o Peer Review è una riunione formale con ruoli precisi:
- MODERATORE -> guida la riunione, media i conflitti, tiene tempo
- AUTORE -> chi ha scritto il codice, è li per chiarire dubbi e perplessita (non è sotto processo, quindi non è li per difendersi)
- REVISORI -> coloro che leggono il codice e cercano difetti
- SCRIBA -> quello che scrive il verbale con i bug trovati, tiene traccia quindi delle informazioni

### Varianti AGILI

- Pair Programming
- Walkthrough / Code Review leggera: quella prima del commit push
- Sprint Review: si giarda il prodotto ottenuto a fine iterazione

## Misurazioni e metriche del Software

### Misurazione VS Metrica

- misurazione -> atto di assegnare un numero ad una proprieta (conto quante righe ha questo file)
- metrica -> risultato standardizzato che usi per fare confronti (questo file ha 5K righe -> troppo lungo)

### Qualita interna vs Esterna

1. interna (strutturale) -> riguarda il codice statico
    - si misura con analisi statica quindi senza eseguire il programma
    - cosa guardi -> modularita coesione accoppiamento
2. esterna (percepita) -> riguarda il codice mentre gira
    - si misura utilizzandolo
    - cosa guardi -> usabilita, performance, affidabilita

### Metriche statiche da sapere

- LOC lines of code
- Complessita Ciclomatica -> quanto è incasinata la logica (quanti if, while, for). Se alta il codice è difficile da mantenere
- Fan-In, Fan-Out -> misurano le dipendenze
    - Fan-In: quanti moduli chiamano me
    - Fan-Out: quanti moduli chiamo io
- Fog Index -> indice di leggibilita (bassa foschia + leggibile, alta foschia - leggibile)

### Metriche OO suite CK

in programmazione ad oggetti si usano metriche __Chidamber & Kemerer CK__:
- WMC weighted methods per class -> quanto è pesante una classe
- DIT depth of inheritance tree -> quanto è profonda l ereditarietà
- CBO coupling between objects -> quanto accoppiamento ce tra le classi
